# 公式カリキュラムツリーPDF確認メモ

2026年度版 `Ver.1.0` の実装で使用した公式PDF確認メモ。

確認日: 2026-06-13

対象PDF:

https://www.nagoya-bunri.ac.jp/wp/wp-content/uploads/2026/04/%E3%82%AB%E3%83%AA%E3%82%AD%E3%83%A5%E3%83%A9%E3%83%A0%EF%BC%88%E6%83%85%E5%A0%B1%EF%BC%89.pdf

補助資料:

- `2026前期情報メディア学科-1.pdf`
- `2024後期情報メディア学科時間割４年ver4.pdf`
- `R8履修の手引き.pdf`

## PDFの性質

- 2026/4版の情報メディア学科カリキュラムツリー
- A3横、2ページ構成
- 1ページ目: 基礎教育科目
- 2ページ目: 専門教育科目、教職課程に関する科目
- PDFはコピー不可設定のため、実装時は画像として目視確認しながら構造化する必要がある

補助資料の位置づけ:

- カリキュラムツリーPDFは、科目間接続、系列、配当年次、レベルの正本とする
- 履修の手引きPDFは、教育課程表と要件文言の補助根拠とする
- 時間割PDFは、既存資料だけでは不足する実際の開講時期や年度内配置の補助根拠として使う
- ただし時間割PDFは年度差の影響を受けるため、将来年度まで固定の開講保証としては扱わない

## `R8履修の手引き.pdf` の参照方針

`R8履修の手引き.pdf` は全学科を含むため、目次を参照して情報メディア学科関連の章だけを利用する。

利用対象:

- 1. 教育課程
- 2. 単位制度（情報メディア学科）
- 3. 卒業・進級要件（情報メディア学科）
- 4. 履修（情報メディア学科）
- 情報メディア学部 情報メディア学科 教職課程の履修について

冊子ページとPDF内部ページにはずれがある。今回確認できた対応は次のとおり。

| 冊子ページ | 内容 | PDFページ |
| :---: | :--- | :---: |
| 78 | 教育課程 | 42 |
| 91 | 卒業・進級要件 | 55 |
| 92-93 | 履修 | 56-57 |
| 110 | 教職課程 | 74 |

このPDFから採る情報は、主に次の範囲に限定する。

- 卒業要件の文言と内訳
- 進級要件の条件
- 履修登録上限と登録上の制約
- 教職課程の基礎資格と必要単位

逆に、他学科向けの教育課程表や全学共通の冊子本文は、このプロジェクトの実装根拠としては採用しない。

## `R8履修の手引き.pdf` から補強できる実装ルール

- 卒業要件:
  - 基礎教育科目 36単位以上
  - 専門教育科目 36単位以上
  - 「卒業に必要なその他の科目」52単位以上
  - 合計 124単位以上
- 基礎教育科目の選択超過分は、「卒業に必要なその他の科目」に 10単位まで算入
- 他学科履修科目は 12単位まで
- 他大学科目は 30単位まで
- 3年次進級条件:
  - 卒業要件科目 50単位以上
  - かつ `通算GPA 1.0以上` または `2年次年間GPA 1.0以上`
- 各学期の履修登録上限は 24単位
- 上級年次配当科目は履修不可
- `ITパスポート` と `基本情報技術` は、申出により単位認定の対象
- 教職課程の基礎資格として `日本国憲法`、`スポーツⅠ・Ⅱ`、`プラクティカル・イングリッシュⅠ`、`情報リテラシー` が明記されている

## 結論

ツリー表示モードの実装に必要な公式資料は、このPDFで概ね揃っている。

ただし、現在のMD資料および `app.js` の科目データだけでは、公式PDFのツリーを忠実に再現するには不足がある。特に専門教育科目のツリーには、現在のアプリに入っていない科目、系列、矢印、レーン配置が含まれている。

したがって、実装前に公式PDFからツリー表示用データを追加で構造化する必要がある。

## 現在データとの差分・不足

### 1. 専門教育科目の科目数が不足

現在の `app.js` には、コース必修8科目と一部専門選択科目だけが入っている。

公式PDFには、以下のような追加科目がある。

- データサイエンス
- 人工知能
- 基本情報技術
- 情報管理
- ゲームプログラミング
- サウンドプログラミング
- 知識情報学
- モバイル情報論
- メディアアート
- ドキュメンタリー・シナリオ
- CM制作
- 写真技術
- 放送・配信論
- 舞台制作
- サウンドプロダクション
- デジタルファブリケーション
- CAD
- メディア文化論
- ドキュメンタリー演習

このため、公式ツリー表示では追加の専門教育科目マスタが必要になる。

基礎教育科目側でも、以下の年次配当科目が実装・資料の双方で見落とされやすい。

- プラクティカル・イングリッシュⅢ
- プラクティカル・イングリッシュⅣ

### 2. 科目名の確認が必要

現在アプリでは `基本情報技術` を資格認定対象科目として扱っている。

公式PDF上では専門教育科目ツリーに `基本情報技術(※)` と読める科目がある。アプリ上の正式名称も `基本情報技術` とし、旧表記 `基本情報技術者` は別名として吸収する。

### 3. 公式の矢印が既存 `prereqs` より多い

現在の `prereqs` は最低限の前提関係だけを持つ。

公式PDFには、以下のような系列内接続や横断接続がある。

- コース共通から専門演習、卒業演習への接続
- プログラミング系列内の接続
- 映像制作系列内の接続
- サウンド制作系列内の接続
- メディアデザイン系列内の接続
- 教職課程科目の連続配置

複数の前提科目を持つものとして、少なくとも次が確認できる。

- データサイエンス: プログラム演習Ⅰ、情報処理演習
- 音響制作演習: 音響学、聴能・音感演習
- デジタルファブリケーション: ＣＧ基礎、ＣＡＤ

教職課程に関する科目は、単体科目ごとの厳密な前提というより、半期ごとの必修群を順に積み上げていく並びとして残すのが適切。

ツリー表示では、`prereqs` とは別に `treeEdges` のような表示用エッジデータを持つのがよい。

### 4. レイアウト情報が必要

公式PDFは、横軸が配当学年期、縦軸が科目区分・コース・系列という構造になっている。

実装では、各ノードに以下を持たせると公式PDFに近い表示にできる。

- `treePage`
- `treeSection`
- `treeLane`
- `treeColumn`
- `level`
- `courseNumber`
- `isBold`
- `edgeGroup`

## 実装時に実施した整理

1. 公式PDFから専門教育科目のノードを構造化した
2. 現在の `app.js` 科目データとの差分を補った
3. ツリー表示用に `officialTreeNodes` と `officialTreeEdges` を定義した
4. 専門教育科目と基礎教育科目を同じツリー表示モードで扱うようにした
5. 時間割資料と補足資料で、開講時期や表示配置の不足を補った

## 構造化の実施状況

2026-06-16時点で、公式PDFを踏まえたツリー情報を `app.js` に追加した。

- `officialTreeNodes`: 103件
- `officialTreeEdges`: 99件
- 追加専門選択科目: データサイエンス、人工知能、情報管理、ゲームプログラミング、知識情報学、ドキュメンタリー演習など
- 追加教職科目: 教育心理学、特別活動論、教育相談論、特別支援教育、教育実習指導など
- 公式ツリーから漏れていた `情報学概論` をコース共通ノードとして補完した
- 基礎教育科目は `treeMetaByName` で46科目分の領域・番号・表示列を補っている
- 教職課程科目は、半期ごとの必修群間を全結合で表示する

補足:

- 公式PDF上の `基本情報技術(※)` に合わせ、履修データ上も `基本情報技術` を正式名称とする
- 旧表記 `基本情報技術者` は別名として `基本情報技術` に正規化する
- 専門教育科目の番号は正式確定値ではなく、プロジェクト内で用いる暫定番号を含む
- `情報学概論` など、公式ツリー上で確認できても正式番号未確認のものは暫定番号を割り当てて空欄を避ける
