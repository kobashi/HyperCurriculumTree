# カリキュラムツリー表示モード 仕様メモ

## 概要

現行の履修計画チェッカーに、カリキュラムツリーを視覚的に表示するモードを追加する。

ツリー表示モードでは、全科目をノードとして画面上に配置し、科目間の接続、コース別のまとまり、履修済み/未履修の状態を直感的に確認できるようにする。ユーザーはツリー上の科目ノードをクリックし、現在の科目一覧と同じ感覚で履修時期を選択して履修計画へ追加できる。

## 目的

- 科目間の前後関係を視覚的に理解しやすくする
- コースごとの科目配置を把握しやすくする
- 4年間の履修計画とカリキュラム全体の関係を同時に見られるようにする
- 科目一覧から探すだけでなく、ツリーから履修科目を選べるようにする

## 対象ユーザー体験

ユーザーは通常の履修計画モードとツリー表示モードを切り替えられる。

ツリー表示モードでは、すべての科目ノードが画面に配置されている。ノードには科目名、単位、区分、配当年次/期、履修状態が表示される。ノードをクリックすると履修時期の選択UIが表示され、選択すると現行の履修計画と同じ `state.planned` に反映される。

## 表示要件

- すべての科目をノードとして表示する
- 科目区分ごとに見分けやすい色やタグを使う
- コース必修は選択中コースと他コースで区別する
- 専門選択は通常の専門選択として表示する
- ITパスポート、基本情報技術は専門選択かつ上限外科目として表示する
- 教職科目は教職課程ON/OFFと連動する
- 履修済みノードは達成状態が分かるようにする
- 未履修ノードは追加可能であることが分かるようにする
- 前提科目や系列がある場合はエッジで接続する

## 操作要件

- ノードクリックで履修時期の選択UIを表示する
- 選択UIは現行の科目カードと同じ選択肢を使う
- 選択後は履修計画グリッド、単位集計、達成状況に即時反映する
- すでに履修済みのノードはクリックで履修時期の変更ができる
- 履修解除もできる
- ツリー表示モードでも標準配置、クリア、コース切り替え、GPA条件、教職課程の挙動は現行と一致させる

## データ要件

既存の `allCourses` をツリー表示の一次データとして使う。

追加で必要になる可能性があるデータ:

- ノードの表示グループ
- ノードの表示順
- 系列情報
- 前提科目、後続科目の明示的な関係
- コース別レーン
- ツリー表示用の補助ラベル

既存資料から取得できる関係:

- `prereqs` に定義済みの前提科目
- `Curriculumtree.md` の系列説明
- コース別必修科目
- 共通必修科目
- 基礎教育科目の前後関係

公式PDF確認後の追加メモ:

- 2026/4版の公式PDFには、現在のMD資料および `app.js` より詳細なツリー配置、科目番号、レベル、系列、矢印が含まれている
- 特に専門教育科目は、現行アプリに未収録の専門選択科目が多い
- 基礎教育科目も、必修/選択の二分だけではなく、PDF上の領域区分と「配当年次無し」列を持つ
- 実装前に `docs/official-curriculum-pdf-review.md` の確認結果を反映し、ツリー表示用の `treeNodes` / `treeEdges` を追加する
- 履修判定用の `prereqs` と、視覚表示用の公式矢印 `treeEdges` は分けて扱う
- 専門教育科目の番号は、正式資料が揃うまでは暫定番号を表示に用いる

## レイアウト案

初期実装では、複雑な自動レイアウトよりも安定した手動/半自動レイアウトを優先する。

候補:

- 横軸: 学年または配当時期
- 縦軸: 科目区分、コース、系列
- エッジ: 前提関係や系列接続

例:

- 基礎教育
- 専門共通
- 情報システム
- 映像メディア
- サウンド制作
- メディアデザイン
- 専門選択
- 他学科
- 教職

## 実装方針

### 1. 表示モード切り替え

現行の履修計画画面に、一覧/ツリーの表示切り替えを追加する。

- 一覧モード: 現在の科目カードUI
- ツリーモード: ノード表示UI

### 2. ノードデータ生成

`allCourses` からツリー用ノード配列を生成する。

ノードに持たせる情報:

- course id
- 科目名
- 単位
- カテゴリ
- コース
- 配当年次
- 開講期
- 履修状態
- CAP対象/上限外
- 教職必修
- 資格取得対象

### 3. エッジデータ生成

`prereqs` と資料内の系列情報からエッジを生成する。

初期実装では `prereqs` 由来のエッジを優先し、系列情報は後続で拡張する。

2026-06-15時点の実装準備:

- `app.js` に公式PDF由来の `officialTreeNodes` を追加
- `app.js` に公式PDF由来の `officialTreeEdges` を追加
- `officialTreeNodes` は履修可能科目の `courseName` を参照し、表示上の公式表記が異なる場合は `displayName` を持つ
- `officialTreeEdges` は履修判定ではなく表示専用の矢印として扱う
- 公式ツリーノード100件、表示用エッジ72件を定義済み
- `情報学概論` のように公式ツリー上の存在が確認できる共通科目は、表示ノードも明示的に持つ
- `courseNumber` は、基礎教育では確認済み番号を優先し、専門教育では未確認分に暫定番号を割り当てる
- 基礎教育科目は `treeMetaByName` により、領域区分と「配当年次なし」列を含めてPDF寄りに表示する
- ノードが参照する科目はすべて `allCourses` に対応済み

### 4. ノード操作

ノードクリック時に履修時期選択パネルを表示する。

選択処理は現行の `setPlanned(course.id, value)` を使い、判定ロジックを二重化しない。

### 5. 状態同期

ツリー表示モードと一覧モードは同じ状態を使う。

- `state.planned`
- `state.course`
- `state.teacher`
- GPA条件
- フィルター条件

## 初期実装の完了条件

- ツリー表示モードに切り替えられる
- 全科目がノードとして表示される
- コース別の配置が視覚的に分かる
- 前提関係のある科目が線で接続される
- ノードクリックで履修時期を選択できる
- 選択結果が現行の履修計画グリッドと集計に反映される
- GitHub Pages上で操作確認できる

## 後回しにすること

- 完全な自動グラフレイアウト
- ドラッグによるノード配置変更
- ノード位置の保存
- 複雑なズーム/パン操作
- 全資料からの系列情報の完全自動抽出

## リスクと注意点

- ノード数が多いため、画面密度が高くなりすぎる可能性がある
- スマートフォン幅ではツリーが見づらくなる可能性がある
- 前提関係と系列関係を混ぜると線が多くなりすぎる可能性がある
- 現行の履修判定ロジックを複製すると不整合が起きるため、状態更新と判定は既存関数を使う
- コース必修と専門選択の重複表示ルールをツリーでも統一する必要がある

## 確認方法

確認はGitHub Pagesのリモート環境で行う。

公開URL:

https://kobashi.github.io/HyperCurriculumTree/

確認項目:

- ツリー表示モードへ切り替えられる
- ノードが表示される
- ノードから履修時期を選べる
- 選択後に履修計画と単位集計が更新される
- 標準配置、クリア、コース変更、教職ON/OFFと矛盾しない
