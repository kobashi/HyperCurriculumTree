# 2026年度版 Ver.1.0 仕様・演出意図まとめ

この文書は、HyperCurriculumTree 2026年度版 `Ver.1.0` から `Ver.1.03.4` の到達点を、今後の保守と同種プロジェクトの参考にするために整理したものです。

## 位置づけ

- 対象: 名古屋文理大学 情報メディア学科 2026年度カリキュラム
- 公開タイトル: 超履修計画2026
- 公開形態: ビルド不要の静的Webアプリ
- 公開先: GitHub Pages
- 開発正本: `main`
- 試作版退避: `prototype/main-before-tree`
- 演出版: `codex/playful-ux` で試作後、`main` に統合済み

Ver.1.0 では、履修計画、卒業要件判定、カリキュラムツリー表示、UI演出、SE/BGM、BGM可視化を含めた版を正本とした。Ver.1.01 では、BGM再生の頭出し条件と音声処理負荷を調整した。Ver.1.02 では、科目一覧の多段グルーピングと配色の統一を行った。Ver.1.03 では、科目リストとツリー表示の配色意味を揃え、前提不足の再帰確認を強化した。Ver.1.03.1 では、公式ツリーの専門科目エッジと履修判定用前提関係の差分を埋めるBug Fixを行った。Ver.1.03.2 では、`全必修` ボタンが既存の履修済み科目を外してしまう不具合を修正した。Ver.1.03.3 では、科目サマリーからの履修時期変更・解除をリスト/ツリーと揃えた。Ver.1.03.4 では、その他52単位の内訳を要件どおりに再整理した。

## 基本仕様

### 履修計画

- 4年間8学期の履修計画を作成する。
- 実際の曜日・時限の時間割は作成しない。
- 科目ごとに履修時期を選択し、履修済み状態として扱う。
- 集中講義、資格取得による単位認定、教職課程、他学科履修も履修計画上の選択肢として扱う。
- リスト表示とツリー表示のどちらでも、科目の色は同じ意味で扱う。通常の未履修は所属カテゴリ/コースの淡色、履修済みは同系統の濃色、必修未履修は赤、前提不足は紫とする。
- 赤は「履修が必要」、紫は「履修が望ましい」を表すため、同じ科目が両方に該当する場合は赤を優先する。
- 前提不足は、複数の前提科目と、未履修前提科目のさらに前提科目まで再帰的に確認する。

### 卒業要件判定

- 卒業要件124単位を、基礎教育、専門教育、その他に分けて確認する。
- 基礎教育選択の上限、他学科履修の上限、要件外単位を区別する。
- その他52単位には、基礎選択の振替10単位まで、専門教育科目、単位認定、他学科科目12単位まで、他大学認定30単位までを含め、専門教育科目36単位へ算入した分を除く分もその他枠に算入する。
- 専門教育は、専門共通必修とコース必修を卒業要件上の専門教育単位として判定する。
- 専門科目全体の内訳は、コース共通、情報システム、映像メディア、サウンド制作、メディアデザインの色分けで表示する。
- 複数コースにまたがる専門選択科目は、内訳表示では該当コースへ按分し、重複カウントしない。

### CAP判定

- 各学期のCAP対象単位と上限外単位を分けて表示する。
- 集中講義、資格取得による単位認定、教職課程科目はCAP上限外として扱う。
- GPA条件は数値入力ではなく、`1.0未満`、`1.0以上`、`3.7以上` のメニューで選ぶ。

### コース選択

- コースは展開メニューから選択する。
- `未選択` はコース未選択状態に戻す。
- `コース変更＋必修追加` は、既存の履修登録を残しながら選択コースのコース必修を追加する。
- `コース変更＋他コース解除` は、選択コースへ変更し、他コースの科目登録を解除する。
- コース共通科目は解除対象にしない。
- 複数コースの科目を履修して卒業要件を満たす運用を妨げないため、追加型の操作を基本にする。

### 一括登録

- `登録` 行に `全必修` と `全削除` を置く。
- `全必修` は基礎必修、専門共通必修、選択中コースのコース必修、教職課程ON時の教職必修を標準配置する。
- 選択科目は自動登録せず、ユーザーが卒業要件を満たすように選ぶ。
- `全削除` は履修登録を空にする。

## ツリー表示仕様

- 公式カリキュラムツリーPDFを正本とする。
- 時間割資料、履修の手引き、補足資料は、開講時期や資料不足を補う補助資料として扱う。
- ツリー表示は科目選択パネルの表示モードであり、リスト表示と同じ `state.planned` を使う。
- ノードは公式PDFに寄せて、基礎教育、コース共通、各コース、教職課程を配置する。
- 科目間の接続は表示用エッジとして管理し、履修判定用の前提関係と混ぜない。
- コース必修科目は、コース未選択時も各コース内に表示し、枠線で強調する。
- 教職課程科目は半期ごとの必修群として順に積み上がる構造を表示する。
- 他学科の表示は絞り込みメニューで制御し、ツリー専用の他学科ON/OFFは持たない。
- 科目番号、補足情報はツリー表示時のみ切り替え可能とし、デフォルトは非表示にする。
- スマートフォンではツリーを無理に画面幅へ縮小せず、必要な横幅を確保して科目名が読めることを優先する。

## UI方針

- 初期画面は実用画面とし、ランディングページは作らない。
- 絞り込み、コース選択、GPA条件、演出設定は、常時表示せず展開式メニューにする。
- 画面上の短縮表記には、必要に応じてホバー説明を付ける。
- 判定パネルでは、要件外、振替、基礎振替、他学科、単位認定、基礎選択超過、他学科超過などの用語説明をホバーで確認できる。
- 達成/未達は色で区別するが、上限外単位そのものを赤で強調しない。赤は上限超過や必修欠損など、対応が必要な状態に使う。
- 進捗バーは、卒業要件全体、基礎教育、専門教育、その他を分けて表示し、内訳が色で分かるようにする。

## 演出意図

演出版は、基本機能を変えずに「履修計画を操作している手応え」を強めるために導入した。ゲーム的な演出は、実用性を壊さない範囲で、状態変化を分かりやすくするための補助として扱う。

### アニメーション

- ツリー表示へ切り替えた時は、根本から順に展開されるような登場演出を入れる。
- 科目の登録、解除、フィルターによる表示/非表示、全必修、全削除では、対象科目が順に変化するように見せる。
- 一括処理でも状態更新を不安定にしないため、演出用の遅延データと実際の履修状態を分ける。
- ゴースト表示や二重クリックが必要になる不具合を避けるため、削除演出は既存DOMへ一時クラスを付け、完了後に通常描画へ戻す。

### SE

- SEは登録、解除、切り替え、エラーなどの操作感を補強する。
- BGMとSEは別バスに分け、SEが鳴ってもBGMを止めない。
- 一括処理ではSEが過剰に重ならないよう、鳴らす回数を制限する。

### BGM

- BGMはコース未選択と4コースで別profileを持つ。
- コースごとにテンポ、スケール、コード進行、リズム、ベースライン、音色、エフェクトを変える。
- 同じコース、同じ履修科目集合では同じBGMになるよう、履修状態から決まるkey/seedを使う。
- 選択科目の追加・削除は、16ステップ内のどこかに小さな音楽的変化として累積する。
- 必修欠損や前提不足は、赤/紫の視覚状態と合わせて、外れ値アレンジや専用ハーモニーとしてBGMに反映する。
- 欠損が16件を超える場合は1箇所へ集中させず、ローリングしながら複数ステップへ重ねる。

### BGM可視化

- ピアノロール風パネルで、旋律、低音、和音、律動、外音を確認できる。
- 通常、微調整、外れ値を色分けし、履修状態のどこがBGMへ反映されたかを確認できる。
- 標準では非表示とし、必要な時だけ演出設定メニューから表示する。

### 音声安定化

- 発音の立ち上がりと終端を丸め、クリックノイズを抑える。
- ノイズ音はバッファの先頭と末尾をフェードさせる。
- BGM/SEの出力段には軽いローパスとリミッターを置く。
- 欠損時の過激なノイズ層は、演出意図を残しつつ、スマートフォンの小型スピーカーでも破綻しにくい量へ抑える。

### Ver.1.01 音声処理調整

- BGMの頭出しはコース変更時だけに限定する。
- 科目追加、科目削除、全必修、全削除、教職ON/OFFでは、BGMを止めずに履修状態のアレンジを継続中の次ステップから反映する。
- BGM解析結果は履修状態ごとにキャッシュし、状態が変わらない間は毎ステップ再計算しない。
- seed付きノイズ音はAudioBufferをキャッシュし、同じ音を生成し直さない。
- 発音終了後のAudioNodeは明示的にdisconnectし、長時間ループで接続が残らないようにする。
- BGM tickは例外が発生しても次のtickを予約し、1ステップの失敗でBGM全体が停止しないようにする。

### Ver.1.03 配色・前提確認調整

- 科目リストの分類色はカード本体へ集約し、補足情報タグには余分な分類色を付けない。
- ツリー表示のノードにも科目リストと同じ配色ルールを適用する。
- 専門選択科目が複数コースのツリーに現れる場合、リスト表示では代表所属を定めて色のずれを防ぐ。ツリー表示では配置されているコースの色を優先する。
- 前提不足は再帰的に収集するが、表示上は必修未履修の赤を優先する。赤表示の科目であっても、その科目の前提確認は継続する。

### Ver.1.03.1 前提関係Bug Fix

- 公式ツリーの専門科目エッジと `prereqs` を照合し、履修判定側に漏れていた前提関係を補った。
- `暮しとデザイン` は、`マルチメディア`、`デジタルファブリケーション`、`Ｗｅｂ解析` を前提として扱う。
- `マルチメディア` の前提探索が `メディアデザイン入門` まで到達するよう、`グラフィックデザイン` と `メディアデザイン入門` の関係を補った。
- `情報メディア特別演習Ⅱ` は `情報メディア特別演習Ⅰ` を前提として扱う。

### Ver.1.03.2 全必修Bug Fix

- `全必修` は未履修の必修科目だけを標準配置し、既存の履修済み科目を外さないようにした。
- コース選択済みなら、そのコースのコース必修だけを追加する。

### Ver.1.03.3 サマリー操作調整

- 科目サマリーの各科目をクリックすると、リスト表示やツリー表示と同じ履修時期修正・解除メニューを開けるようにした。

### Ver.1.03.4 その他要件Bug Fix

- 判定の「その他52単位」に入る枠を、基礎選択の振替10単位まで、専門教育36単位から外れる専門教育科目、他学科12単位まで、他大学認定30単位までに整理した。

## 資料・実装方針

- PDF、時間割、履修の手引き、補足資料はリポジトリ内で管理する。
- Markdown資料は、実装に使うための再構造化資料として扱う。
- 実装で見つけた資料不足や読み取り修正は、アプリだけでなくMarkdown資料へ戻す。
- 専門教育科目の番号は、シラバス検索システムで個別確認しないと完全確定できないため、Ver.1.0では暫定番号を割り当てて運用する。

## 検証項目

- `node --check app.js`
- `git diff --check`
- リスト表示/ツリー表示の切り替え
- 科目登録、解除、履修時期変更
- 全必修、全削除
- コース変更＋必修追加
- コース変更＋他コース解除
- 教職課程ON/OFF
- 資格取得と科目履修の排他
- 絞り込み表示/非表示
- 卒業要件、CAP、要件外単位、専門科目内訳
- SE/BGM ON/OFF、BGM可視化ON/OFF
- Pages公開URLに `?v=<commit>` を付けた確認

## 既知の制約

- 実時間割の曜日・時限衝突は扱わない。
- ツリー表示はスマートフォンでは横幅を必要とする。
- 専門教育科目番号は暫定値を含む。
- 公式資料、補助資料、担当教員補足が食い違う場合は、ドキュメントに根拠を残したうえで実装判断する。
- 音声演出はブラウザの自動再生制限により、初回ユーザー操作後に有効になる。
